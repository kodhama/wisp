# .grove/ — wisp's grove dials

This directory holds what wisp owns of its grove install (`adr-0026`, the
thin-vendor boundary): the gate-profile, the shared role config, and this
dial-explainer. The agent roles themselves are **plugin-carried** — they load
as `grove:<role>` subagents from the installed grove plugin and are never
vendored into this repo. This file is grove-managed (regenerated by
`/grove:refresh`). Effect of each dial:

- **`gates.toml`** — yours: who must approve at each of the four gates
  (`intent` / `spec` / `build` / `ship`), `human` or `agent`. The floor
  (`adr-0018`): at least one human intent-locus gate — `intent = human` OR
  `ship = human` — validated on every run-sequencing read. wisp's profile is
  `steward` (intent + ship human; spec + build agent). Switch presets with
  `/grove:set-profile <preset>` or hand-edit a single row.
- **`config.toml`** — yours: the shared role tokens (`adr-0026` D3) grove
  agents resolve at use time — test/typecheck commands, parked-item store,
  rubric paths, corpus dirs. Values are **verified priors**: agents verify
  before relying, disclose mismatches loudly, and route fixes back here rather
  than silently working around a stale value. "None exists yet" is an honest
  value.
- **`agents/<role>.md`** — yours, optional: per-role addenda (local rules,
  worked examples) a generic `grove:<role>` agent reads when present and
  replaces with judgment when absent. wisp carries one: `run-resumer`
  (branch-finding convention).
- **`internal/`** — grove's: the gate-profile floor-guard machinery
  (`internal/gates/`) and the C1 enforcement defaults
  (`internal/enforcement.toml`), re-copied verbatim on refresh; not a consumer
  surface — don't hand-edit.

**Version stamp.** wisp pins the fleet at `grove plugin@0.1.0`, recorded in the
AGENTS.md managed block (`adr-0026` D4). At role start, an agent whose installed
grove version differs from that stamp discloses the skew loudly and continues —
the stamp is the in-repo ratified record, never a lock; grove never enforces it.

The operating model itself (lifecycle enum, versioning grammar, edge taxonomy)
is **not restated here** (`adr-0008`): it ships in the grove plugin — per the
grove lifecycle / versioning / relations companions, `plugin@0.1.0` (the stamp
in this repo's AGENTS.md).
