Four pieces, one story

What ships

01

Protocol

The functional core: event schema, parse, and a team-state reducer. Tested in isolation — protocol.ts, covered by test/protocol.test.ts.

02

Bus

The file transport: append-only NDJSON at .grove/runtime/events.ndjson (override with $GROVE_EVENTS). Runtime state is gitignored — telemetry is a self-reported claim, never artifact-derived truth.

03

Emitter

The agent-facing CLI — status, heartbeat, verdict, question, plus command check/ack. What an agent calls to report itself, nothing more.

04

Dashboard

A zero-dep HTTP shell (dashboard + /api/state + /api/events + POST /api/command) serving a self-contained UI: swarm graph, agent cards, state timeline, open questions answerable from the page, raw event feed.

05

Demo furrow

demo.ts — a synthetic furrow replay, modeled on the Grove prototype's furrow 1, that feeds the bus so the dashboard is watchable without a real swarm. It's what the hero above is built from.

06

Zero dependencies

Node ≥22.18, run directly via type stripping. No build step, no package manager required, nothing installed underneath it.

Install

Vendor it

wisp ships as source, not a package, today. The family's current distribution default is vendored-copy: clone the repo, or copy its flat files straight into your project.

$ git clone https://github.com/kodhama/wisp.git
$ cd wisp && node server.ts # dashboard on :4177
$ cp path/to/wisp/{protocol,bus,emit,server}.ts path/to/wisp/dashboard.html your-project/
$ node server.ts # no install step — plain node, nothing underneath it

Publishing @kodhama/wisp to npm (the name is already reserved in package.json) is a later option, not a commitment this page makes — vendored-copy is today's default.
